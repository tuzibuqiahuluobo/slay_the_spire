# 尖塔之魂 - 卡牌 Roguelike (Python 版)

这是一款融入轻度肉鸽元素的单机卡牌构建游戏，玩家需要通过构筑卡组、利用召唤物机制与敌人展开回合制战斗。

## 核心特色机制

1. **召唤物体系**：玩家可以召唤“小星星”作为伙伴。它可以协同攻击敌人，同时也可以为玩家抵挡伤害。
2. **Buff/Debuff 与虚化机制**：包含力量提升、虚弱减伤以及无视伤害的虚化状态。
3. **流畅的拖拽打牌体验**：实现了平滑的卡牌拖拽曲线以及丰富的浮动伤害和特效反馈。

## 模块化项目架构

本游戏的 Python 版本采用模块化设计，主要分为以下目录：

- `main.py`: 游戏主入口，初始化并驱动游戏循环。
- `settings.py`: 全局常量（分辨率、颜色、帧率等）。
- `core/`: 游戏逻辑与数据核心。包括 `game_state.py` (状态机)、`entity.py` (实体类) 以及 `cards.py`。
- `ui/`: 表现层。主要处理分辨率缩放适配 (`display_manager.py`)、视图渲染 (`battle_scene.py`, `card_view.py`) 以及特效动画 (`fx_manager.py`)。
- `utils/`: 工具箱。处理输入映射、碰撞检测及一些通用函数。
- `assets/`: 游戏所有美术与声音素材的放置区。

## UI 素材替换说明

如果需要替换或增加界面 UI 素材，请按照以下规范放入 `assets/images/`，以保证在游戏内的尺寸不突兀：

- `entities/` (角色/怪物/召唤物原画)：
  - **建议尺寸**: `200x200` 至 `300x300`，透明背景 PNG
  - 小星星尺寸: `100x100` 至 `150x150`
- `icons/` (意图、Buff 图标):
  - **建议尺寸**: `64x64`，透明背景 PNG
- `cards/` (卡牌背景或元素):
  - **建议尺寸**: `250x350` (比例 5:7 左右)
  - 能量图标: `100x100`

## 多分辨率适配机制

本游戏采用 **Resolution Independent (分辨率独立)** 的设计思想：
- 所有内部逻辑运算和坐标绘制基于 **1920x1080**。
- 最终通过 `ui/display_manager.py` 将画面等比例缩放至物理窗口尺寸（支持从最低 `800x600` 到最高 `2560x1440`）。
- 鼠标点击事件也会通过 `utils/input_handler.py` 逆向映射至逻辑分辨率，方便未来扩展至触摸平台（移动端）。
